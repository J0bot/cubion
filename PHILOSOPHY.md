# Cubion Philosophy

This document is not a legal document. It's a statement of intent.  
The MIT license gives you full freedom to do what you want with this project.  
This file explains what we *want* — and why.

---

## Backward compatibility is a moral commitment

A cube built in 2027 must work with a cube built in 2127.  
Not because we can legally force you to — we can't.  
Because if you break compatibility, you're making someone's hardware useless.  
That person trusted the standard. Don't break that trust.

The CFC connector spec is frozen at v1.0 not to limit you —  
but to give everyone a stable foundation to build on forever.

---

## Open source, always

Cubion started open source. It should stay that way.  
You can build proprietary cubes. You can sell them. That's fine.  
But if you define a new connector standard and call it "Cubion-compatible" —  
make the spec public. The ecosystem only works if everyone can participate.

---

## Design for longevity, not for growth hacking

The goal is not to ship fast and iterate.  
The goal is to get the foundation right so we never have to break it.  
Slow down on the spec. Speed up on the implementation.

A standard that lasts 1000 years is worth more than a product that ships next quarter.

---

## Decentralization is not optional

There is no "main" cube. No server. No cloud dependency.  
A cluster of cubes in a basement must work with zero internet, zero registration, zero account.  
This is not a feature — it's a design constraint.

---

## Inspired by

- **Linux** — because it just works, everywhere, for decades  
- **USB** — because one standard connected the entire world  
- **Minecraft** — because simple primitives unlock infinite creativity  
- **Xerboxion** — because technology should serve individual sovereignty
